# Mkdocs によるサイト閲覧・PDF出力

<!-- 
次は今作成したMarkdown のファイルをブラウザで快適に閲覧できるようにします。
Markdown はテキストファイルなので GitHub でも見ることができますが、探しにくいという欠点があります。

そこで、静的サイトジェネレーターの MkDocs を採用することにしました。
MkDocs を使うことで、検索機能、サイドバーの目次、テーマの統一、そして PDF への一括変換など、ドキュメント閲覧体験を大幅に向上させることができます。

実際にやっていきます。

まず、mkdocs.yml ファイルを開きます。
これはmkdocsの設定が書かれているファイルです。

すでにmkdocsの初期設定は終わっていますので、ここに先ほど追加したファイルに対するナビゲーションを追加します。
それが終わったらターミナルで「mkdocs serve」を入力して、実行します。

「mkdocs serve」を実行すると、Markdown ファイルが HTML に変換され、自分の PC がローカル開発サーバーになります。
ブラウザで http://127.0.0.1:8000/ にアクセスすると、変換されたサイトが表示されます。

先ほど追加した デモ 用のページが追加されていることがわかります。
ファイルを編集するとリアルタイムで反映されるので、ドキュメント作成中にブラウザで確認しながら作業できます。
-->

# ソースとドキュメントの一元管理
