# ドキュメント記述マニュアル

本プロジェクトのドキュメント記述ルール。textlint設定に基づく。

## 文体

- である調への統一（見出し・本文・箇条書きすべて）
- 箇条書きは体言止め

## 文章構成

- 1文あたり最大150文字
- 1文あたり読点最大4個
- 同一助詞の連続使用禁止（「も」「や」「か」は許容）
- 同一接続詞の連続使用禁止
- 二重否定禁止

## スペース

- 全角・半角間スペースなし
- インラインコード前後スペースなし

## 句読点・記号

- 文末は句点
- 文中区切りは読点
- ピリオド・カンマは使用禁止
- 感嘆符・疑問符は全角
- 丸かっこ・大かっこは全角

## 数字

- 数量・計測値は算用数字
- 慣用句・固有名詞は漢数字

## 表記ゆれ

### 長音

語尾が-er、-or、-arの外来語には長音付与（サーバー、ユーザー、ブラウザー、フォルダー）

### ひらく漢字

以下はひらがな表記：
できる、さらに、すべて、あらかじめ、ほとんど

### 技術用語

正式表記を使用（JavaScript、TypeScript、GitHub）

## 検証コマンド

```bash
pnpm lint:text
pnpm lint:text:fix
```
